About Login Screen:

The login screen is the page where the functionality starts.

Here there are multiple options:

  • Login:

    A registered user can login from here. His username and password will go back to the server through the service and check the authenitcation through PassportJS. If the user is authenticated, a new session is created and the user data is returned.

    Here is a screen shot of the login screen:

  • Forgot Password:

    The user may also recover his password. He has 2 options. Either he can ask for the system to e-mail the password or you can reset the password by answering the security questions.

    For using either of the options, the user need to verify his Email Id after registering.

    Screen shot:

    Recover password using email

    Recovering password through email, requires the user to enter his username and click the button. A mail will be sent giving his password back.

    The system will show an error when a user who did not verify his email tries to recover.

    Screen shot:

    Recover password using security question

    The second method of recovering a forgotten password is by resetting the password after answering the security question.

    Here when the user enters his username and clicks the button, he gets the security question.

    Screen shot:

    The user should answer the security question correctly to reset his password, else an error message is displayed.

    Screen shot:

    When ever a user changes his password, a e-mail will be sent saying they the password was changed.

  • Register:

    The user must enter all the fields to get registered.

    When the user registers, a verification link will be sent to the email. On clicking on this link, the user's email will be verified.

    On registering, the system automatically logs him in and creates a session.

    The registration page looks like this:

  • Go For Free:

    This is for a user who do not wish to register into the system, but still wishes to use the system.

    A free user does not have access to all the features of the system. For example, he can not use the email widget or he can not save his current position of the widgets.

  • Admin Login:

    The admin logs into the system by clicking on the admin login button at the bottom left corener of the login screen.

    He gets the login screen as shown below:

    The username and password goes to the server to check the authentication and comes bac with success or error message.

    If the details are correct, he logs into the system, else an error message is displayed.

  • Lock Screen:

    The user can go back to the lock screen from the login screen by clicking on the icon at the bottom right corner of the screen.

Implementation:

All the popups in the screen are developed using the ngDialog.

All the data goes to the server through the service and comes back with a success or error message.

The passportJs and express sessions are used for handling authentication and sessions.
